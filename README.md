# nanopods-house

A Next.js 16 app that serves the ArchCraft architecture-studio demo (originally a WordPress + Elementor site) as the visual frontend.

## Architecture in 30 seconds

The site is **legacy HTML, served through Next.js**. Static HTML files live under `static_site/demo.casethemes.net/archcraft/` (one folder per route). A single Next.js route handler at [`src/app/[[...slug]]/route.ts`](src/app/[[...slug]]/route.ts) catches every URL and returns the matching `index.html` verbatim. Asset paths under `/wp-content/*` resolve via the `public/wp-content` symlink â€” Next serves them directly without hitting the route handler.

The header and footer have been extracted into [`static_site/.../partials/`](static_site/demo.casethemes.net/archcraft/partials/) (`header.html`, `footer.html`, `components.js`) and are loaded client-side by every page via `<div data-component="header"></div>` placeholders. To change the header or footer once for all pages, edit the partial.

## Quickstart

```bash
npm install
npm run dev          # Next.js on http://localhost:3000

# (Optional) static reference for visual diffing on a different port:
npx serve static_site/demo.casethemes.net/archcraft -l 3000 --no-clipboard
npm run dev -- -p 3001
```

## Editing pages

- **Header / Footer:** edit `static_site/.../partials/header.html` or `partials/footer.html` once. Every page picks it up.
- **A single page:** edit the corresponding `static_site/.../<route>/index.html`. The HTML is one giant line per file (auto-generated by WordPress); use `python` regex scripts rather than hand-editing.
- **Graduating a page to React:** create `src/app/<route>/page.tsx` â€” Next.js routes to it instead of the catch-all. Rebuild that route's UI with whatever components you want; nothing else changes.

## Known intentional gaps

- WooCommerce pages (`/shop`, `/cart`, `/checkout`, `/wishlist`, `/my-account`, `/product/...`) render demo content but have no backend.
- Newsletter and contact forms POST to `#` â€” no real backend.
- A few WordPress XHRs (`wp-json/contact-form-7/...`, `wc-ajax=woosw_get_data`) 404 silently and don't affect the page.
- `Squada One` font is referenced in legacy CSS but never loaded; falls back to Albert Sans.
