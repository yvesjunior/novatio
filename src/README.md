# nanopods-house

A Next.js 16 app that serves the Novatio architecture-studio demo (originally a WordPress + Elementor site) as the visual frontend.

## Architecture in 30 seconds

The site is **legacy HTML, served through Next.js**. Static HTML files live under `static_site/archcraft/` (one folder per route). A single Next.js route handler at [`src/app/[[...slug]]/route.ts`](src/app/[[...slug]]/route.ts) catches every URL and returns the matching `index.html` verbatim. Asset paths under `/wp-content/*` resolve via the `public/wp-content` symlink — Next serves them directly without hitting the route handler.

The header and footer have been extracted into [`static_site/archcraft/partials/`](static_site/archcraft/partials/) (`header.html`, `footer.html`, `components.js`) and are loaded client-side by every page via `<div data-component="header"></div>` placeholders. To change the header or footer once for all pages, edit the partial.

## Quickstart

```bash
npm install
npm run dev          # Next.js on http://localhost:3000
```

## Project layout

```
src/app/
  [[...slug]]/route.ts    catch-all: every URL → static_site/<slug>/index.html
  layout.tsx              minimal HTML wrapper
  globals.css             Tailwind v4 import (unused by served pages today)

static_site/archcraft/
  home-two/               → /
  about-us/, contact-us/, blog-grid/, services/, faq/, ...   → matching URLs
  partials/               shared header / footer (single source of truth — inlined at build time)
  wp-content/             theme + plugin CSS / JS / uploads

public/
  wp-content              symlink → static_site/archcraft/wp-content (so /wp-content/* works under Next)

tests/                    Playwright visual regression
```

## Editing pages

- **Header / Footer:** edit `static_site/archcraft/partials/header.html` or `partials/footer.html`. The next `npm run dev` / `npm run build` runs `scripts/build-partials.mjs` which stamps the partials into every HTML file (idempotent — wraps inlined content in `<!-- partial:NAME -->...<!-- /partial:NAME -->` so re-runs swap the latest content). To stamp on demand: `npm run build:partials`.
- **A single page:** edit the corresponding `static_site/archcraft/<route>/index.html`. The HTML is one giant line per file (auto-generated by WordPress); use `python` regex scripts rather than hand-editing.
- **Graduating a page to React:** create `src/app/<route>/page.tsx` — Next.js routes to it instead of the catch-all. Rebuild that route's UI with whatever components you want; nothing else changes.

## Known intentional gaps

- WooCommerce pages (`/shop`, `/cart`, `/checkout`, `/wishlist`, `/my-account`, `/product/...`) render demo content but have no backend.
- Newsletter and contact forms POST to `#` — no real backend.
- A few WordPress XHRs (`wp-json/contact-form-7/...`, `wc-ajax=woosw_get_data`) 404 silently and don't affect the page.
- `Squada One` font is referenced in legacy CSS but never loaded; falls back to Albert Sans.

## Lead-qualification chatbot

A floating chat widget appears bottom-right on every page ([public/chatbot.js](public/chatbot.js)). It walks a visitor through 5 scripted questions, classifies them as **HOT / WARM / COLD**, and POSTs the result to [`/api/lead`](src/app/api/lead/route.ts).

Leads always go to the server console and append to `leads.json` (gitignored). To also notify a real destination, copy `.env.example` to `.env.local` and set one or more of:

- `LEAD_SLACK_WEBHOOK_URL` — Slack incoming-webhook URL
- `LEAD_RESEND_API_KEY` + `LEAD_FROM_EMAIL` + `LEAD_TO_EMAIL` — email via [Resend](https://resend.com)
- `LEAD_GENERIC_WEBHOOK_URL` — any URL that accepts the lead JSON (Discord webhook, Zapier, n8n, custom CRM)

All channels fire in parallel; one failing doesn't block the others. Edit [`src/app/api/lead/route.ts`](src/app/api/lead/route.ts) to change classification rules, message format, or add new channels.
